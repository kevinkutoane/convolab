# Ubiquitous Language Guide for ConvoLab

This document defines the core terminology used within the ConvoLab platform, ensuring a shared understanding across all stakeholders—developers, product owners, and domain experts. Adhering to this ubiquitous language is crucial for maintaining consistency, clarity, and precision in communication and within the codebase.

## Core Concepts

### Conversation

*   **Definition**: A continuous exchange of messages between a user and the ConvoLab platform, potentially involving multiple turns and interactions with various AI capabilities.
*   **Key Attributes**: `ConversationId`, `UserId`, `TenantId`, `Messages`, `Status`.
*   **Context**: Represents the state and history of an ongoing dialogue.

### Workflow

*   **Definition**: A predefined, structured sequence of steps or operations that ConvoLab executes to achieve a specific outcome, often in response to a user request or system event. Workflows orchestrate the interaction between different engines (Prompt, Knowledge, AI, Evaluation, Trace).
*   **Key Attributes**: `WorkflowId`, `Name`, `Description`, `Version`, `Nodes`, `Transitions`.
*   **Distinction**: Replaces the previous concept of 
`Pipeline` to better reflect enterprise workflow orchestration.

### Execution

*   **Definition**: A runtime instance of a `Workflow`, representing the actual process of executing a defined sequence of steps. It manages the lifecycle, state transitions, and outcomes of a specific workflow run.
*   **Key Attributes**: `ExecutionId`, `WorkflowVersionId`, `Status`, `StartTime`, `EndTime`, `OverallResult`, `ExecutionContext`.
*   **Responsibilities**: Lifecycle management, scheduling, workflow progression, state transitions, retry policies, cancellation, timeouts.

### Trace

*   **Definition**: A comprehensive record of a single request or operation as it flows through the ConvoLab system, encompassing all related `Spans`, `Events`, `Metrics`, and `Artifacts`. It provides end-to-end visibility for debugging, performance analysis, and auditing.
*   **Key Attributes**: `TraceId`, `CorrelationId`, `OperationName`, `StartTime`, `EndTime`, `Status`, `Spans`.
*   **Model**: Designed as a distributed tracing model, supporting nested spans and correlation IDs for future OpenTelemetry mapping.

### Prompt

*   **Definition**: The input provided to an AI model, typically consisting of instructions, context, and examples, designed to elicit a specific response. In ConvoLab, this often refers to a `PromptTemplate` that is prepared with dynamic data before AI invocation.
*   **Key Attributes**: `PromptTemplateId`, `Content`, `Parameters`, `Type`.
*   **Engine**: `IPromptEngine` is responsible for preparing and rendering prompts.

### Knowledge

*   **Definition**: Information retrieved from various sources (e.g., databases, documents, external APIs) that provides context or factual data to enhance AI responses or workflow execution.
*   **Key Attributes**: `KnowledgeBaseId`, `Name`, `Description`, `KnowledgeItem`.
*   **Engine**: `IKnowledgeEngine` is responsible for searching and retrieving relevant knowledge.

### Evaluation

*   **Definition**: The process of assessing the quality, accuracy, and relevance of AI responses or overall workflow outcomes against predefined criteria or metrics.
*   **Key Attributes**: `EvaluationId`, `Report`, `Metrics`, `Result`, `Status`.
*   **Engine**: `IEvaluationEngine` performs the assessment and generates reports.

### Provider

*   **Definition**: An external service or system that offers AI capabilities (e.g., OpenAI, Anthropic, Gemini) or other essential services (e.g., vector databases). ConvoLab interacts with providers through abstract interfaces.
*   **Key Attributes**: `Name`, `Endpoint`, `IsActive`, `Health`.
*   **Abstraction**: `IAIOrchestrationService` abstracts AI provider interactions.

### Model

*   **Definition**: A specific AI model offered by a `Provider` (e.g., `gpt-4`, `claude-3-opus`, `gemini-1.5-pro`). Each model has distinct `Capabilities` and `Availability`.
*   **Key Attributes**: `AIModelId`, `Name`, `ProviderName`, `Capabilities`, `Availability`.

### Completion

*   **Definition**: The process of generating text or other content by an AI model based on a given `Prompt` or `Messages`. Also refers to the generated output itself.
*   **Key Attributes**: `Content`, `TokenUsage`, `FinishReason`, `Cost`.

### Embedding

*   **Definition**: A numerical representation (vector) of text or other data, capturing its semantic meaning. Used for tasks like semantic search, recommendation, and clustering.
*   **Key Attributes**: `Vector`, `Dimensions`, `TokenUsage`.

### Plugin

*   **Definition**: An extensible component that adds specific functionality or integrates with external tools/services within a workflow. Plugins allow ConvoLab to perform actions beyond its core capabilities.
*   **Key Attributes**: `PluginId`, `Name`, `Version`, `ManifestUrl`, `Status`.
*   **Manager**: `IPluginManager` handles the registration and invocation of plugins.

### Tool

*   **Definition**: A specific function or capability exposed by a `Plugin` that an AI model can invoke to perform actions or retrieve information. This is often referred to as "function calling" in AI contexts.
*   **Key Attributes**: `ToolName`, `Arguments`.

### Memory

*   **Definition**: The mechanism by which ConvoLab retains and utilizes past conversational context or other relevant information across turns or sessions to maintain coherence and provide more informed responses.
*   **Reference**: `MemoryReference` in `ExecutionContext` points to the location or identifier of the stored memory.

using ConvoLab.Domain.Common;
using ConvoLab.Domain.Conversation.ValueObjects;
using ConvoLab.Domain.Users.ValueObjects;
using ConvoLab.Domain.Prompt.ValueObjects;
using ConvoLab.Domain.Knowledge.ValueObjects;
using ConvoLab.Domain.AI.ValueObjects;

namespace ConvoLab.Domain.Execution.ValueObjects;

public class ExecutionContext : ValueObject
{
    public ExecutionId ExecutionId { get; private set; }
    public ConversationId? ConversationId { get; private set; }
    public Guid? WorkflowId { get; private set; }
    public Guid? TenantId { get; private set; }
    public UserId? UserId { get; private set; }
    public Guid CorrelationId { get; private set; }
    
    public string? Culture { get; private set; }
    public string? Locale { get; private set; }
    public string? Timezone { get; private set; }
    
    public IReadOnlyList<string> FeatureFlags { get; private set; }
    public string? SelectedProvider { get; private set; }
    public AIModelId? SelectedModel { get; private set; }
    
    public IReadOnlyDictionary<string, string> ExecutionVariables { get; private set; }
    public string? MemoryReference { get; private set; }
    public string? PromptReference { get; private set; }
    public string? KnowledgeReference { get; private set; }
    
    public IReadOnlyDictionary<string, string> Metadata { get; private set; }
    public IReadOnlyList<string> Attachments { get; private set; }
    
    public DateTime ExecutionStartTime { get; private set; }
    public DateTime? ExecutionDeadline { get; private set; }

    private ExecutionContext(
        ExecutionId executionId,
        Guid correlationId,
        ConversationId? conversationId = null,
        Guid? workflowId = null,
        Guid? tenantId = null,
        UserId? userId = null,
        string? culture = null,
        string? locale = null,
        string? timezone = null,
        IEnumerable<string>? featureFlags = null,
        string? selectedProvider = null,
        AIModelId? selectedModel = null,
        IReadOnlyDictionary<string, string>? executionVariables = null,
        string? memoryReference = null,
        string? promptReference = null,
        string? knowledgeReference = null,
        IReadOnlyDictionary<string, string>? metadata = null,
        IEnumerable<string>? attachments = null,
        DateTime? executionStartTime = null,
        DateTime? executionDeadline = null)
    {
        ExecutionId = executionId;
        CorrelationId = correlationId;
        ConversationId = conversationId;
        WorkflowId = workflowId;
        TenantId = tenantId;
        UserId = userId;
        Culture = culture;
        Locale = locale;
        Timezone = timezone;
        FeatureFlags = featureFlags?.ToList().AsReadOnly() ?? new List<string>().AsReadOnly();
        SelectedProvider = selectedProvider;
        SelectedModel = selectedModel;
        ExecutionVariables = executionVariables ?? new Dictionary<string, string>();
        MemoryReference = memoryReference;
        PromptReference = promptReference;
        KnowledgeReference = knowledgeReference;
        Metadata = metadata ?? new Dictionary<string, string>();
        Attachments = attachments?.ToList().AsReadOnly() ?? new List<string>().AsReadOnly();
        ExecutionStartTime = executionStartTime ?? DateTime.UtcNow;
        ExecutionDeadline = executionDeadline;
    }

    public static ExecutionContext Create(ExecutionId executionId, Guid correlationId)
    {
        return new ExecutionContext(executionId, correlationId);
    }

    // Wither patterns for immutability
    public ExecutionContext WithConversationId(ConversationId conversationId) => 
        new(ExecutionId, CorrelationId, conversationId, WorkflowId, TenantId, UserId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, SelectedModel, ExecutionVariables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    public ExecutionContext WithWorkflowId(Guid workflowId) => 
        new(ExecutionId, CorrelationId, ConversationId, workflowId, TenantId, UserId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, SelectedModel, ExecutionVariables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    public ExecutionContext WithTenantId(Guid tenantId) => 
        new(ExecutionId, CorrelationId, ConversationId, WorkflowId, tenantId, UserId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, SelectedModel, ExecutionVariables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    public ExecutionContext WithUserId(UserId userId) => 
        new(ExecutionId, CorrelationId, ConversationId, WorkflowId, TenantId, userId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, SelectedModel, ExecutionVariables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    public ExecutionContext WithSelectedModel(AIModelId modelId) => 
        new(ExecutionId, CorrelationId, ConversationId, WorkflowId, TenantId, UserId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, modelId, ExecutionVariables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    public ExecutionContext WithVariables(IReadOnlyDictionary<string, string> variables) => 
        new(ExecutionId, CorrelationId, ConversationId, WorkflowId, TenantId, UserId, Culture, Locale, Timezone, FeatureFlags, SelectedProvider, SelectedModel, variables, MemoryReference, PromptReference, KnowledgeReference, Metadata, Attachments, ExecutionStartTime, ExecutionDeadline);

    protected override IEnumerable<object> GetEqualityComponents()
    {
        yield return ExecutionId;
        yield return CorrelationId;
        if (ConversationId != null) yield return ConversationId;
        if (WorkflowId != null) yield return WorkflowId;
        if (TenantId != null) yield return TenantId;
        if (UserId != null) yield return UserId;
    }

    // For EF Core
    private ExecutionContext() { 
        ExecutionId = null!;
        FeatureFlags = new List<string>().AsReadOnly();
        ExecutionVariables = new Dictionary<string, string>();
        Metadata = new Dictionary<string, string>();
        Attachments = new List<string>().AsReadOnly();
    }
}

### AI Orchestration
*   **Definition**: The process of dynamically selecting, configuring, and invoking appropriate AI models and providers based on the `ExecutionContext` and `WorkflowDefinition`.
*   **Service**: `IAIOrchestrationService` encapsulates this logic.

### AI Provider
*   **Definition**: An external service (e.g., OpenAI, Anthropic, Gemini) that offers AI capabilities. ConvoLab interacts with providers through abstract interfaces.
*   **Key Attributes**: `Name`, `Endpoint`, `AuthenticationDetails`, `RateLimits`.

### AI Model
*   **Definition**: A specific AI model offered by an `AI Provider` (e.g., `gpt-4`, `claude-3-opus`, `gemini-1.5-pro`). Each model has distinct `Capabilities` and `Availability`.
*   **Key Attributes**: `AIModelId`, `Name`, `ProviderName`, `Capabilities`, `Availability`, `MaxTokens`, `Pricing`.

### Model Capability
*   **Definition**: A specific feature or function supported by an `AI Model` (e.g., `TextCompletion`, `ChatCompletion`, `Embedding`, `ImageGeneration`, `ToolCalling`). Represented as a Flags enum.

### Model Availability
*   **Definition**: The operational status and accessibility of an `AI Model` (e.g., `Online`, `Offline`, `Degraded`, `Maintenance`).

### Completion Request/Response
*   **Definition**: The data structure used to send a request to an AI model for text completion or chat completion, and the corresponding response received from the model.
*   **Key Components**: `Prompt`, `Messages`, `Temperature`, `MaxTokens`, `StopSequences` (Request); `Content`, `TokenUsage`, `FinishReason`, `AICost` (Response).

### Embedding Request/Response
*   **Definition**: The data structure used to send a request to an AI model for generating embeddings, and the corresponding response containing the numerical vector representation.
*   **Key Components**: `InputText`, `Model` (Request); `Vector`, `Dimensions`, `TokenUsage` (Response).

### Token Usage
*   **Definition**: A metric indicating the number of tokens consumed by an AI model for a given request, typically broken down into `PromptTokens` and `CompletionTokens`.

### AI Cost
*   **Definition**: The monetary cost associated with an AI model invocation, calculated based on `TokenUsage` and the model's pricing.

### Workflow Definition
*   **Definition**: The blueprint or template of a `Workflow`, defining its structure, sequence of `WorkflowNode`s, and configuration. It is versioned to allow for iterative changes.
*   **Key Attributes**: `WorkflowDefinitionId`, `Name`, `Description`, `LatestVersionId`.

### Workflow Version
*   **Definition**: A specific, immutable version of a `WorkflowDefinition`, ensuring that `WorkflowExecution` instances can always refer to a consistent workflow structure.
*   **Key Attributes**: `WorkflowVersionId`, `WorkflowDefinitionId`, `VersionNumber`, `Nodes`, `Edges`, `CreatedAt`.

### Workflow Node
*   **Definition**: A single step or task within a `WorkflowVersion`. Each node represents a specific operation (e.g., `PromptGeneration`, `KnowledgeRetrieval`, `AIInvocation`, `Evaluation`).
*   **Key Attributes**: `NodeId`, `NodeType`, `Configuration`, `InputMappings`, `OutputMappings`.

### Workflow Execution
*   **Definition**: A runtime instance of a specific `WorkflowVersion`, tracking its progress, state, and results. It manages the state transitions of the workflow through a state machine.
*   **Key Attributes**: `ExecutionId`, `WorkflowVersionId`, `Status`, `StartTime`, `EndTime`, `CurrentNodeId`, `ExecutionContext`.

### Workflow State Machine
*   **Definition**: The mechanism within `WorkflowExecution` that governs valid state transitions (e.g., `Pending` -> `Running` -> `Completed` or `Failed`). It enforces business rules and emits domain events upon state changes.

### Execution Context
*   **Definition**: An immutable value object that carries all necessary runtime information and parameters throughout a `WorkflowExecution`. It acts as a central data bag for the workflow.
*   **Key Attributes**: `ExecutionId`, `ConversationId`, `WorkflowId`, `TenantId`, `UserId`, `CorrelationId`, `Culture`, `Locale`, `Timezone`, `FeatureFlags`, `SelectedProvider`, `SelectedModel`, `ExecutionVariables`, `MemoryReference`, `PromptReference`, `KnowledgeReference`, `Metadata`, `Attachments`, `ExecutionStartTime`, `ExecutionDeadline`.

### Span
*   **Definition**: A logical unit of work within a `Trace`, representing an operation or a step in a distributed system. Spans can be nested to show parent-child relationships.
*   **Key Attributes**: `SpanId`, `TraceId`, `ParentSpanId`, `OperationName`, `StartTime`, `EndTime`, `Status`, `Attributes`.

### Trace Event
*   **Definition**: A specific event that occurs within a `Span`, providing additional context or detail about the operation.
*   **Key Attributes**: `Timestamp`, `Name`, `Attributes`.

### Metric
*   **Definition**: A numerical measurement captured during a `Span` to quantify performance or behavior (e.g., duration, token count, cost).
*   **Key Attributes**: `Name`, `Value`, `Unit`, `Timestamp`.

### Artifact
*   **Definition**: A piece of data or output associated with a `Span` or `Trace` (e.g., a generated prompt, an AI response, an error log).
*   **Key Attributes**: `Name`, `Type`, `Content`, `MimeType`.

### Tool Call
*   **Definition**: An instruction from an AI model to invoke a specific `Tool` (function) provided by a `Plugin`.
*   **Key Attributes**: `ToolName`, `Arguments` (JSON).

### Memory Reference
*   **Definition**: A pointer or identifier within the `ExecutionContext` to a specific memory store or conversational history, allowing the workflow to access and update long-term context.

### Prompt Reference
*   **Definition**: A pointer or identifier within the `ExecutionContext` to a specific `PromptTemplate` or generated prompt content.

### Knowledge Reference
*   **Definition**: A pointer or identifier within the `ExecutionContext` to retrieved knowledge items or a `KnowledgeBase`.

### Tenant
*   **Definition**: A distinct customer or organizational unit within the ConvoLab platform, each with its own isolated data, configurations, and users.
*   **Key Attributes**: `TenantId`, `Name`, `SubscriptionPlan`.

### User
*   **Definition**: An individual interacting with the ConvoLab platform.
*   **Key Attributes**: `UserId`, `Email`, `Roles`, `Preferences`.

### Feature Flag
*   **Definition**: A mechanism to enable or disable features dynamically without deploying new code, allowing for A/B testing, phased rollouts, and personalized experiences.
*   **Usage**: `FeatureFlags` in `ExecutionContext` can influence workflow behavior.

### Correlation ID
*   **Definition**: A unique identifier that links all operations and events related to a single logical request across different services and systems, crucial for distributed tracing and logging.
*   **Usage**: Included in `ExecutionContext` and `Trace`.
